# Session Handover — 2026-04-13

## What Was Built (This Session)

Two major features on `feature/ws-agent-protocol`, merged to main (32 commits):

### 1. WebSocket Agent Protocol
AI agents control the game via semantic commands over WebSocket.

- **Godot:** GameActions autoload (action bus), WSClient, S_ActionProcessor
- **Server:** Mastra-powered Node.js server with 4 tools (move, interact, switch_era, get_state)
- **Protocol:** JSON messages (command, ack, state_event, state_snapshot, handshake)
- **Verified:** GLM-5.1 agent successfully played the game (move, observe, reason about enemies)

### 2. Replay System
Sessions recorded to SurrealDB with per-turn vector embeddings for agent learning.

- **SurrealDB:** 3 tables (replay_session, replay_event, replay_turn) with HNSW vector index
- **fastembed:** BGE-small-en-v1.5 (384 dims) for per-turn embeddings
- **Recorder:** Passive observer on ConnectionManager
- **Context Builder:** Queries session summaries + vector-searched similar turns
- **Verified:** Replay context grows across runs (155 → 519 → 892 chars), agent receives past experience

### 3. Infrastructure
- Provider rotation: Z.AI (GLM-5.1) + Alibaba (Qwen 3.5-plus) round-robin
- `infra/.env` for API keys (gitignored), `infra/.env.example` committed
- JSON schemas for WS messages, game state, replay records

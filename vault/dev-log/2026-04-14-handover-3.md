## Bugs Fixed During Verification

### 1. PCK loading blocked by COEP (Cross-Origin-Embedder-Policy)
**Root cause:** Static web server sends `Cross-Origin-Embedder-Policy: require-corp`, which blocks cross-origin resources without `Cross-Origin-Resource-Policy` header. The PCK fetch from game server (different port = cross-origin) was silently blocked.
**Fix:** Added `Cross-Origin-Resource-Policy: cross-origin` header to game server PCK responses, plus CORS preflight (OPTIONS) handler.

### 2. _process() runs before _ready() completes in web builds
**Root cause:** Godot web builds — `_ready()` uses `await` for PCK HTTP fetch, but `_process()` starts running immediately without waiting. Entities/views not yet created → null reference errors.
**Fix:** Added `_ready_complete` flag guard at top of `_process()`.

### 3. GDScript type inference errors with `:=` and const operations
**Root cause:** Godot 4.6's type inference fails when using `:=` with expressions involving `const` floats (e.g., `var x := sz.y * MONSTER_ZONE`). Works in editor but fails in export.
**Fix:** Replaced all `:=` with explicit type annotations (`var x: float = ...`) and changed `const FOO :=` to `const FOO =`.
